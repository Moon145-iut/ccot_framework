# Running the CCOT Thesis Framework

This project operates as a five-stage pipeline. Follow the sections below in order; every command is Windows- and PowerShell-friendly, but works the same in bash (replace `python` with `python3` if needed).

## 1. Environment Setup
```bat
cd ccot_thesis_framework
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```
Copy `.env.example` to `.env` and fill in any API keys you plan to use (`GEMINI_API_KEY`, `OPENAI_API_KEY`, `OPENAI_BASE_URL`).

## 2. Prepare Teacher Signals
Place your GSM8K CSV somewhere (must contain `question` and `answer` columns). Convert it into a JSONL of teacher rationales + answers:
```bat
python -m ccot.cli prepare-teacher --csv data\gsm8k_train.csv --out artifacts\teacher.jsonl
```
Optional debugging: add `--limit 32` to process a subset.

**Using an API teacher:**
```bat
python -m ccot.cli prepare-teacher --csv data\gsm8k_train.csv --out artifacts\teacher_api.jsonl ^
  --provider gemini --teacher-model gemini-1.5-pro --use-api-for-rationale
```
Replace `--provider` with `openai_compat` and add `--base-url http://localhost:8000/v1` for vLLM/SGLang/Qwen servers.

## 3. Build Local Hidden Targets
This stage loads a HuggingFace causal LM (CPU) and extracts hidden states from question/reasoning/answer segments:
```bat
python -m ccot.cli build-targets --teacher-jsonl artifacts\teacher.jsonl --out-dir artifacts\targets ^
  --model-id Qwen/Qwen2.5-0.5B-Instruct --compression-ratio 0.1 --selector evenly_spaced
```
Key outputs:
- `targets.npz`: q-features + padded latent tensors + lengths + ids
- `samples.jsonl`: metadata per sample
- `meta.json`: backbone + selector configuration (used later by inference)

## 4. Train the CCOT Latent Generator
```bat
python -m ccot.cli train-ccot --targets-dir artifacts\targets --out-dir artifacts\ccot_weights ^
  --epochs 25 --batch-size 4 --lr 1e-3
```
Saves `latent_generator.pt` and `train_log.json` inside the output directory.

## 5. Train the Char-Level Decoder
```bat
python -m ccot.cli train-decoder --targets-dir artifacts\targets ^
  --ccot-weights artifacts\ccot_weights\latent_generator.pt ^
  --out-dir artifacts\decoder_weights --epochs 30 --batch-size 8
```
By default this step conditions on latents generated by the trained CCOT model. Add `--use-teacher-latents` to train against oracle teacher latents.

## 6. Local Inference
Provide a custom math question and run the locally trained models end-to-end:
```bat
python -m ccot.cli infer --question "Lena buys 2 apples after owning 3. How many apples?" ^
  --targets-dir artifacts\targets ^
  --ccot-weights artifacts\ccot_weights\latent_generator.pt ^
  --decoder-weights artifacts\decoder_weights\char_decoder.pt
```
The CLI prints JSON containing the predicted answer string and the number of latents produced.

## 7. Demo Script
`run_demo.bat` strings together every command with conservative epochs/batch sizes. Edit the CSV/model paths inside the script, then run it from the project root:
```bat
run_demo.bat
```

## 8. Tips & Troubleshooting
- Use `--num-threads N` on long-running commands to control CPU usage.
- Increase `--max-length` during `build-targets` if your concatenated segments exceed the default 1024 tokens.
- API failures automatically fall back to dataset rationales with diagnostic labels in `teacher_source`.
- Delete/replace artifacts between experiments to avoid mixing incompatible metadata (particularly `meta.json`).
